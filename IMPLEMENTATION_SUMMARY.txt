================================================================================
                          DATABASE SCHEMA
================================================================================

TABLE: career_applications

Columns:
- id (UUID) - Primary key, auto-generated
- first_name (TEXT) - Required
- last_name (TEXT) - Required
- email (TEXT) - Required, indexed
- phone (TEXT) - Required
- job_title (TEXT) - Required, indexed
- experience (INTEGER) - Required
- resume_url (TEXT) - Required
- cover_letter (TEXT) - Optional, max 1000 chars
- linkedin (TEXT) - Optional
- github (TEXT) - Optional
- portfolio (TEXT) - Optional
- applied_at (TIMESTAMP) - Auto-set to current time

Indexes:
- idx_career_applications_email (on email)
- idx_career_applications_job_title (on job_title)

Row-Level Security:
- Public can insert applications
- Authenticated users can view their own applications

================================================================================
                            API ENDPOINT
================================================================================

Endpoint: POST /api/CareerApplication
Content-Type: application/json

Request Body:
{
  "firstName": "John",
  "lastName": "Doe",
  "email": "john@example.com",
  "phone": "+919876543210",
  "jobTitle": "Senior Software Engineer",
  "experience": "5",
  "resumeUrl": "https://drive.google.com/file/...",
  "coverLetter": "I am interested in...",
  "linkedin": "https://linkedin.com/in/john",
  "github": "https://github.com/john",
  "portfolio": "https://johndoe.com"
}

Response (Success):
{
  "message": "Application submitted successfully",
  "data": {
    "id": "uuid",
    "first_name": "John",
    ...
  }
}

Response (Error):
{
  "message": "Error description"
}

Status Codes:
- 200: Success
- 400: Missing required fields
- 500: Server error

================================================================================
                         TROUBLESHOOTING
================================================================================

ISSUE: Form submission fails
SOLUTION: 
- Check career_applications table exists
- Verify RLS policies are set
- Check Supabase credentials
- Look at browser console for errors

ISSUE: Country codes not showing
SOLUTION:
- Hard refresh browser (Cmd+Shift+R on Mac)
- Clear browser cache
- Check JavaScript console

ISSUE: Styling looks off
SOLUTION:
- Clear .next folder: rm -rf .next
- Restart dev server
- Check Tailwind CSS is loaded

ISSUE: Data not saving to database
SOLUTION:
- Verify .env.local variables
- Check table permissions in Supabase
- Ensure RLS "Allow public to insert" policy exists
